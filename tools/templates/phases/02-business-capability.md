# Phase 2 · Business & Capability

**Purpose.** Capabilities, value streams, processes, organization. **ADM analogue:** B. **Agentic addition:** **capability allocation** (human/agent/hybrid) and the **automation-frontier map** as primary outputs; roles gain performer-type. This is where the central AEAF design act — capability allocation — happens. (→ Book 1 §9, §11; Book 2 §8, §17.3.)

## Inputs
- Vision, ambition, risk envelope; the autonomy scale.

## Steps
1. Map the capability model, value streams, processes, organization; name capabilities **outcome-first and actor-agnostic**.
2. Record the **capability allocation** for each in-scope capability: human / agent / hybrid team. **[+I]**
3. Produce the **Capability Automation-Frontier Map**: current human/agent boundary and its projected direction. **[+I]**
4. Give roles a **performer-type** (human / agent / hybrid). **[+I]**
5. Name a **accountability owner** for each agent-allocated or hybrid capability. **[+I]**
6. Decompose decisions and fill the **Human-Agent RACI** + initial **Trust & Accountability Matrix** + per-capability **autonomy assignment**.

## Produces
- `business-capability-catalog` (with performer mix) · `capability-automation-frontier-map` · `human-agent-raci` · `autonomy-level-classification` Part B · `trust-accountability-matrix` (initial) · `role-catalog`, `organization-actor-catalog` updates.

## Exit gate (must be true before Phase 3)
- [ ] Capability model mapped; capabilities outcome-first and actor-agnostic
- [ ] Capability allocation recorded per in-scope capability **[+I]**
- [ ] Capability Automation-Frontier Map produced **[+I]**
- [ ] Roles carry a performer-type **[+I]**
- [ ] Each agent/hybrid capability has a named accountability owner **[+I]**
