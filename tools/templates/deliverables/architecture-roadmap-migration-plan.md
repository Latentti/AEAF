# Architecture Roadmap & Migration Plan

**Deliverable.** The Phase 7–8 document that sequences how the target architecture is reached. In AEAF the sequencing rule is **trust earned**: agents pilot under tight oversight first; autonomy widens only as assurance accrues. Each step is a **frontier move** with a trigger and an assurance precondition.

## Sections

| Section | What to record / assemble |
|---|---|
| **Baseline → target** | Where the frontier is now and where it is going (→ `capability-automation-frontier-map`). |
| **Frontier moves** *(+I)* | Each intended re-allocation: from level X to Y, its **trigger** and **assurance precondition**. |
| **Autonomy-graduation gates** | The evidence each move requires (→ `autonomy-level-classification`). |
| **Work packages** | The eval-suite builds, agent rollouts, platform work, sequenced. |
| **Human transition** | Reskilling and role change for people whose work moves to agents. |
| **Dependencies & risk** | Sequencing constraints; the business case. |
| **Sign-off** | Programme owner + architecture board. |

**Assembled from** the Frontier Map, the Autonomy-Level Classification, and (for a retrofit) the gap register. A roadmap arrow is an *intention with a precondition*, never an authorisation — the gate authorises the move when the precondition holds.
